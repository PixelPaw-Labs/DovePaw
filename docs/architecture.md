# Architecture

DovePaw is a three-layer runtime. The design has one rule: **edit source, run install, everything else regenerates.**

## Runtime Layers

```
Browser UI (Next.js :7473)
  ↓ SSE
Dove — orchestrator agent (Claude Agent SDK + in-process MCP server)
  ↓ ask_* / start_* / await_* tools
A2A Servers — one Express process per agent (OS-assigned ports)
  ↓ spawn tsx
Agent Scripts — from installed plugin repos, run as launchd daemons
```

### Layer 1 — Browser UI

Next.js chatbot at `localhost:7473`. Streams agent progress, tool calls, and text responses via SSE. The UI is thin — no conversation state, no agent logic. All intelligence lives in the layer below.

### Layer 2 — Dove (Claude Agent SDK)

Dove is a `query()` session from `@anthropic-ai/claude-agent-sdk` — the same runtime that powers Claude Code. An in-process MCP server injects three tools per registered agent (`ask_*`, `start_*`, `await_*`). Dove receives the user's message, decides which agent(s) to invoke, calls the tools, and streams results back to the browser.

Conversation memory is managed by the SDK under `~/.claude/projects/` — not a database. Each turn passes `resume: sessionId` to `query()` to replay from SDK storage.

### Layer 3 — A2A Servers and Agent Scripts

Each agent runs as an independent Express process. The server receives A2A messages from Dove, spawns the agent script (`tsx main.ts`), and streams output back over SSE. The agent script contains the actual logic.

Ports are OS-assigned at startup and published to `~/.dovepaw/.ports.<port>.json`. Dove polls this manifest — no hardcoded ports.

## Key Design Decisions

| Decision                      | Reason                                                                                                                  |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| OS-assigned A2A ports         | No port conflicts, no config to maintain                                                                                |
| Build artifacts are generated | `.mjs` and `.plist` files under `~/.dovepaw/cron/` are always generated by `npm run install` — never edit them directly |
| Plugin system                 | Agents live in external repos; DovePaw clones them at install time — no sensitive code has to go in the DovePaw repo    |
| Dual-mode workflows           | Same `main.ts` handles both chatbot invocation (`argv[2]`) and scheduled batch mode (no arg)                            |
| Dynamic agent registry        | Which agents exist is determined at runtime by which plugins are installed — no hardcoded agent list                    |

## Tech Stack

| Layer                | Technology                                  |
| -------------------- | ------------------------------------------- |
| UI                   | Next.js + React, Tailwind CSS + shadcn/ui   |
| Orchestrator         | `@anthropic-ai/claude-agent-sdk`            |
| Agent protocol       | `@a2a-js/sdk` (SSE)                         |
| Agent runtime        | TypeScript via `tsx`, bundled with `tsup`   |
| Daemon management    | macOS launchd                               |
| Schema validation    | Zod                                         |
| Linting / formatting | oxlint + oxfmt                              |
| Testing              | Vitest                                      |
| Desktop shell        | Electron (menubar + embedded browser panel) |

## File Layout

```
agents/                   → symlink to ~/.dovepaw/plugins/ (all installed plugin agents)
chatbot/
  app/                    → Next.js pages and API routes
  a2a/                    → A2A Express servers (one per agent)
  lib/                    → session store, hooks, progress tracking
lib/                      → shared: agent registry, scheduler, settings, paths
packages/agent-sdk/       → shared agent utilities (Claude runner, logger, git helpers)
scripts/
  build.ts                → compiles agents, links skills, generates launchd plists
electron/                 → Electron shell, menubar app, embedded browser bridge
~/.dovepaw/
  plugins/                → installed plugin repos (cloned by npm run plugin:add)
  plugins.json            → plugin registry
  settings.json           → global settings (repos, API keys)
  settings.agents/        → per-agent config (schedule, env vars, repos)
  workspaces/             → isolated agent execution roots
  cron/                   → compiled daemon scripts (generated by npm run install)
  agent-links.json        → inter-agent handoff graph
```
